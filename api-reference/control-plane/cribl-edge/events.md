## Get events generated by a specified source

<a id="opIdgetEdgeEventsQuery"></a>

> Code samples

`GET /edge/events/query`

Get events generated by a specified source

<h3 id="get-events-generated-by-a-specified-source-parameters">Parameters</h3>

|Name|In|Type|Required|Description|
|---|---|---|---|---|
|Files|query|string|false|Optional parameter, when provided the files and provided offsets are used to look for events.|
|source|query|string|false|The type of source events to search in the format {type}/{subtype} i.e system_state/hostsfile, system_state/routes, etc...|
|limit|query|integer|false|Maximum number of log lines to retrieve starting from offset.|
|et|query|integer|false|Epoch timestamp of the earliest event (includes rolled files present on disk)|
|lt|query|integer|false|Epoch timestamp of the latest event (includes rolled files present on disk)|
|filter|query|string|false|Filter|

> Example responses

> 200 Response

```json
{
  "count": 0,
  "items": [
    {}
  ]
}
```

<h3 id="get-events-generated-by-a-specified-source-responses">Responses</h3>

|Status|Meaning|Description|Schema|
|---|---|---|---|
|200|[OK](https://tools.ietf.org/html/rfc7231#section-6.3.1)|a list of any objects|Inline|
|401|[Unauthorized](https://tools.ietf.org/html/rfc7235#section-3.1)|Unauthorized|None|
|500|[Internal Server Error](https://tools.ietf.org/html/rfc7231#section-6.6.1)|Unexpected error|[Error](#schemaerror)|

<h3 id="get-events-generated-by-a-specified-source-responseschema">Response Schema</h3>

Status Code **200**

|Name|Type|Required|Restrictions|Description|
|---|---|---|---|---|
|» count|integer|false|none|number of items present in the items array|
|» items|[object]|false|none|none|

<aside class="success">
This operation does not require authentication
</aside>
