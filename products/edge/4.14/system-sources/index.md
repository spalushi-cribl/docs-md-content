# System


You can use the following [Sources](sources) to receive data from internal senders.

| Source | Description |
| --- | --- |
| [Exec](sources-exec) | Execute a command periodically, and collect its stdout output |
| [File Monitor](sources-file-monitor) | Collect log files and generate events from the file content |
| [Journal Files](sources-journal-files) | Collects data from systemd's journald service |
| [Kubernetes Events](sources-kubernetes-events) | Generate events for a Kubernetes cluster |
| [Kubernetes Logs](sources-kubernetes-logs) | Collects logs from containers on a Kubernetes node |
| [Kubernetes Metrics](sources-kubernetes-metrics) | Generate events for Kubernetes clusters, nodes, Pods, and containers |
| [System Metrics](sources-system-metrics) | Collect metrics from a host, and populate dashboards |
| [System State](sources-system-state) | Collects the system's current state on a configurable schedule, relaying corresponding events to downstream systems. |
