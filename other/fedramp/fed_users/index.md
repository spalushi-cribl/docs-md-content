Cribl.Cloud Government delivers enterprise-grade access control designed specifically for government agencies with complex organizational structures and strict security requirements. The platform's role-based access control (RBAC) framework ensures proper separation of duties, enables compliance with Federal security mandates, and provides flexible delegation of administrative responsibilities.

The members and teams functionality in Cribl.Cloud Government mirrors the capabilities found in commercial Cribl.Cloud, allowing agencies to leverage existing documentation and training resources.

## User Management

Cribl.Cloud Government's comprehensive user management capabilities help agencies organize and control access at scale.

### Members Management
  * Create and manage individual users with assigned roles and permissions  
  * Import users via SSO with automatic role mapping for efficient onboarding  
  * Configure granular password policies and account security settings  
  * Implement user-specific MFA requirements based on access levels and job functions  
  * Monitor user activity and authentication events for security compliance  

### Teams Administration
  * Create logical groupings of users with shared access requirements  
  * Organize personnel by department, mission function, or project team  
  * Assign collective permissions at the team level for efficient management  
  * Configure team-specific settings and resource access boundaries  
  * Enable hierarchical team structures for complex organizations

For details, see [Members](/stream/members/) and [Teams](/stream/teams/) documentation. 

## Permissions Framework

Cribl.Cloud Government's permission system offers both standard predefined roles and custom roles options.

### Predefined Roles

- **Administrator**: Full platform control for security personnel
- **Standard User**: Day-to-day operational access for analysts
- **Restricted User**: Limited, read-only access for stakeholders

Ready-to-use roles accelerate deployment and ensure consistency across your organization.

### Custom Role Creation

- Design agency-specific roles with precisely defined permissions.
- Define custom access profiles based on specialized job functions.
- Control access to specific resources, data streams, and capabilities.
- Implement least privilege principles for enhanced security posture.
- Create role separation that mirrors your organization's structure.

### Permission Categories
- **System Configuration**: Control over settings and global parameters.
- **Data Management**: Capabilities for collection, processing, and routing.
- **Analytics**: Access to reporting and visualization capabilities.
- **Administration**: Authority over user and role management functions.

For details, see [Permissions](/stream/permissions/).
